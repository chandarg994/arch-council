What is ARCH-COUNCIL?

Plain English

The Problem It Solves

When a business needs an AI solution, someone has to figure out exactly how to build it: what systems, how they connect, what risks exist, how it fits the business. This is AI Solutions Architecture. Normally this takes weeks from expensive specialists. ARCH-COUNCIL does it in minutes.

The Two Ideas It Fuses

llm-council (Karpathy) - Run multiple AI models in parallel on the same problem. Let them review each other anonymously. Have a Chairman synthesise the best answer.

autoresearch (Karpathy) - A ratchet loop: propose improvement, measure quality, keep only if score improves. Repeat until stuck. Only accepts upgrades.

Tech Stack

Every technology, why it was chosen, how it fits

FastAPI (Python)

Async-native backend - critical for running 6 LLM calls simultaneously without blocking. Background tasks let the pipeline run while the browser polls every 3 seconds.

React + Vite

Live-updating frontend showing council opinions, ratchet score progression, and final report. Hot-reload during development. Single-page app polling the backend in real time.

OpenRouter

Single API key accessing Claude, GPT-4o, and Gemini simultaneously. Essential for the multi-model council pattern without managing three separate credentials.

ChromaDB + Sentence Transformers

Vector database storing 17 hand-crafted TOGAF and AI architecture documents. Semantic search retrieves relevant knowledge per agent query. Runs fully locally - no embedding API cost.

Mermaid.js

Each agent includes a Mermaid diagram in their response. The frontend detects these and renders them as interactive flowcharts and deployment diagrams automatically.

Pydantic + JSON Sessions

Each pipeline run is a typed Pydantic model saved as JSON to disk. Simple, portable state - no PostgreSQL or Redis required. UTF-8 atomic writes prevent race conditions.
